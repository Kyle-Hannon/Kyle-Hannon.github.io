Basis Data Structures

Explaining basic data structures

Stacks

Stacks are linear data structures where insertion and removal is done at the "top" of the stack.

  • To define an empty stack use variable_name = Stack().
  • To check if a stack is empty you use variable_name.isEmpty().
  • To add an item to your stack you use variable_name.push(item)
  • To remove the item on top you use variable_name.pop().
  • To check the number of items in the stack you use vairable_name.size().
  • To get the top item without removing it you use variable_name.peak().

Queues

Queues are linear data structures where insertion is done at one end of the stack and removal is done at the other.

  • To define an empty stack use variable_name = Queue().
  • To chekc if a queue is empty you use variable_name.isEmpty().
  • To add an item to your queue you use variable_name.enqueue(item).
  • To remove the item that is on top of your queue you use variable_name.dequeue().
  • To check the number of items in the queue you use variable_name.size().

Dequeues

Dequeues are linear data structures where insertion and removing can happen at both ends.

  • Deque() creates a new deque that is empty. It needs no parameters and returns an empty deque.
  • addFront(item) addsd a new item to the front of the deque. It needs the item and returns nothing.
  • addRear(item) adds a new item to the rear of the deque. it needs the item and returns nothing.
  • removeFront() removes the front item from the deque. It needs no parameters and returns the item. The deque is modified.
  • removeRear() removes the rear item from the deque. It needs no parameters and returns the item. The deque is modified.
  • isEmpty() tests to see whether the deque is empty. It needs no parameters and returns a boolean value.
  • size() returns the number of items in the deque. It needs to parameters and returnss an integer.

Lists

Lists are collections of items where each item holds a relative position with respect to the otherss.

  • List() creates a new list that is empty. It needs no paramaters and returns an empty list.
  • add(item) addsd a new item to the list. It needs theitem and returns nothing. Assume the item is not already in the list.
  • remove(item) removes the item from the list. It needs the item and modifiese the list. Assume the item is present in the list.
  • search(item) searches for the item in the list. It needs the item and returns a boolean value.
  • isEmpty() tests to see wheather the list is empty. It needs no parameters and returns a boolean value.
  • length() returns the number of items in the list. It needs no parameters adn returns an integer.
  • append(item) adds a new items to the end of the list making it the last item in the collection. It needs the item and returns nothing. Assume the item is not already in the list.
  • index(item) returns the position of item in the list. It needs the item and returns the index. Assume the item is in the list.
  • insert(pos, item) adds a new item to hte list at position pos. It needs the item and returns nothing. Assume hte item is not already in the list and ther are enought excisting items to have position pos.
  • pop() removes and returns the last item in the list. It needs nothing and returns an item. Assume the list has atleast one item.
  • pop(pos) removesa and returns the item at position pos. It needs the position and retuns the item. Assume the item is in the list.

Unordered List

Unordered Lists, also known as linked lists or singly linked lists or doubly linked lists, are lists that hold a peice of data with one or two pointers pointing at another peice of data.
